## How It Works
RoPE divides the head dimension into pairs and applies a 2D rotation to each pair: the query at position m is rotated by angle mθ_i, and the key at position n is rotated by nθ_i, where θ_i = 10000^(-2i/d). The dot product q_m · k_n then depends only on (m-n), encoding relative position. This is implemented efficiently as element-wise multiplication with complex exponentials.

## Key Components
The rotation frequencies θ_i decrease geometrically across dimension pairs, similar to sinusoidal encodings. The base value (10000 by default, often increased to 500000 for long context) controls the range of positions the encoding can represent. RoPE is applied inside each attention layer to Q and K, not to the input embeddings.

## Step by Step
For each attention head, split Q and K into pairs of dimensions. Multiply each pair by a complex rotation e^(imθ_i) for queries and e^(inθ_i) for keys. Compute attention scores normally — the relative position is now implicitly encoded in the dot products.
